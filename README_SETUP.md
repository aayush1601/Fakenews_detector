# FakeNewsDB — Setup Guide

## Prerequisites
- Oracle XE 21c installed and running (thin mode — **no Oracle Instant Client needed**)
- Python 3.9+

---

## Step 1 — Install Python dependencies
```bash
pip install -r requirements.txt
```

---

## Step 2 — Create Oracle user and tables

Open SQL*Plus as SYSDBA and create the app user:
```sql
sqlplus sys/yourpassword@localhost:1521/XEPDB1 as sysdba

CREATE USER fakenews IDENTIFIED BY yourpassword;
GRANT CONNECT, RESOURCE, CREATE SESSION TO fakenews;
GRANT CREATE TABLE, CREATE SEQUENCE, CREATE INDEX TO fakenews;
ALTER USER fakenews QUOTA UNLIMITED ON USERS;
EXIT;
```

Then run the DDL script as the fakenews user:
```bash
sqlplus fakenews/yourpassword@localhost:1521/XEPDB1 @create_tables.sql
```

> **Note:** All `id` columns use `GENERATED BY DEFAULT AS IDENTITY` so Oracle
> auto-assigns primary keys when rows are inserted without an explicit id.

---

## Step 3 — Train ML models
```bash
python train_models.py
```

This creates:
- `models/nb.pkl`       — Naive Bayes pipeline
- `models/lr.pkl`       — Logistic Regression pipeline
- `models/bilstm.pkl`   — BiLSTM proxy (char n-gram SGDClassifier)
- `models/bert.pkl`     — BERT proxy (CalibratedLinearSVC)
- `models/vectorizer.pkl` — Shared TF-IDF vectorizer (legacy)
- `train_results.json`  — Accuracy / F1 / Precision / Recall per model
                          (used by load_data.py to populate Oracle tables)

Training takes ~2–5 minutes on the ISOT dataset (~47 k samples).

If Oracle env vars are already set, `train_models.py` will also push the
results directly to the `model_results`, `bias_scorecard`, and
`cross_dataset_performance` tables.

---

## Step 4 — Load data into Oracle
```bash
export ORACLE_USER=fakenews
export ORACLE_PASSWORD=yourpassword
export ORACLE_DSN=localhost:1521/XEPDB1

python load_data.py
```

This loads all 8 tables:

| Table                    | Rows (approx.) | Source                            |
|--------------------------|---------------:|-----------------------------------|
| `isot_dataset`           | ~47 000        | ISOT_Fake.csv + ISOT_True.csv     |
| `liar_dataset`           | ~12 800        | liar_train.tsv + liar_test.tsv    |
| `fnn_pf_dataset`         | ~2 500         | FakeNewsNet_politifact_*.csv      |
| `fnn_gc_dataset`         | ~22 000        | FakeNewsNet_gossipcop_*.csv       |
| `experiments`            | 1              | from train_results.json           |
| `model_results`          | 4              | from train_results.json           |
| `bias_scorecard`         | 4              | from train_results.json           |
| `cross_dataset_performance` | 4           | from train_results.json           |

> Run `train_models.py` **before** `load_data.py` so that `train_results.json`
> is available for populating the analytics tables.

---

## Step 5 — Start the Flask app
```bash
export ORACLE_USER=fakenews
export ORACLE_PASSWORD=yourpassword
export ORACLE_DSN=localhost:1521/XEPDB1

python app.py
```

Open your browser at: **http://localhost:5000**

## Environment Variables

| Variable          | Default               | Description             |
|-------------------|-----------------------|-------------------------|
| `ORACLE_USER`     | `fakenews`            | Oracle schema user      |
| `ORACLE_PASSWORD` | `yourpassword`        | Oracle password         |
| `ORACLE_DSN`      | `localhost:1521/XEPDB1` | Oracle DSN (thin mode) |

All three files (`app.py`, `load_data.py`, `train_models.py`) read the same
env vars so you only need to set them once.

## Oracle SQL Examples

```sql
-- Basic dataset queries
SELECT * FROM isot_dataset FETCH FIRST 10 ROWS ONLY;
SELECT label, COUNT(*) AS cnt FROM isot_dataset GROUP BY label ORDER BY label;

-- ML performance dashboard
SELECT model_name, ROUND(AVG(accuracy),4) AS avg_acc,
       ROUND(AVG(f1_score),4) AS avg_f1
FROM model_results GROUP BY model_name ORDER BY avg_acc DESC;

-- Bias scorecard
SELECT * FROM bias_scorecard ORDER BY disparity_score DESC;

-- Cross-dataset performance
SELECT * FROM cross_dataset_performance ORDER BY accuracy DESC;

-- LIAR dataset — binary label counts
SELECT binary_label, COUNT(*) AS cnt
FROM liar_dataset GROUP BY binary_label;
```

## ML Models

| Model      | Architecture                          | Accuracy (ISOT) |
|------------|---------------------------------------|-----------------|
| NaiveBayes | TF-IDF (1-2g) + MultinomialNB         | ~94%            |
| LogReg     | TF-IDF (1-2g, sublinear) + LR         | ~99%            |
| BiLSTM     | char TF-IDF (2-5g) + SGDClassifier    | ~97%            |
| BERT       | word TF-IDF (1-3g) + CalibratedSVC    | ~99%            |

All models are sklearn pipelines — the vectorizer is embedded inside the `.pkl`
file so no separate tokenisation step is needed at inference time.

## Troubleshooting

**ORA-01017: invalid username/password** — Check `ORACLE_USER` / `ORACLE_PASSWORD` env vars.

**ORA-12541: no listener** — Oracle XE isn't running. Start it with:
```bash
sudo systemctl start oracle-xe-21c   # Linux
# or via Oracle XE startup script on macOS/Windows
```

**FileNotFoundError: models/lr.pkl** — Run `python train_models.py` first.

**Performance dashboard shows no data** — Run `python load_data.py` (after `train_models.py`)
to populate the `model_results` table that the dashboard queries.
